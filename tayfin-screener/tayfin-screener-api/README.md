# tayfin-screener-api

Overview
--------
The `tayfin-screener-api` component exposes HTTP endpoints for querying screener results generated by `tayfin-screener-jobs`. This README documents endpoints, example requests/responses, schema locations, and validation guidance for QA.

Base URL
--------
- Runtime base (example): `http://localhost:8083` — actual port configurable via service env vars.

Endpoints
---------
- GET /v1/screener/{strategy}/latest?ticker=SYMBOL
  - Description: Return the most recent screener result for `ticker` under `strategy`.
  - Query params: `ticker` (required), `as_of_date` (optional, ISO-8601)
  - Response: 200 JSON object or 404 if not found.
  - Schema: `tayfin-screener-api/schemas/screener_result_latest.json`

- GET /v1/screener/{strategy}/range?ticker=SYMBOL&start=YYYY-MM-DD&end=YYYY-MM-DD
  - Description: Return screener results for the ticker in the date range.
  - Response: 200 JSON array.
  - Schema: `tayfin-screener-api/schemas/screener_result_range.json`

- GET /v1/screener/{strategy}/search?tag=TAG
  - Description: Search recent results by tag/label.
  - Response: 200 JSON array.

Example Request
---------------
curl example (local):

```
curl -sS "${TAYFIN_SCREENER_API_BASE_URL:-http://localhost:8083}/v1/screener/vcp/latest?ticker=AAPL" \
  -H "Accept: application/json"
```

Example Response (200)
----------------------
{
  "ticker": "AAPL",
  "strategy": "vcp",
  "as_of_date": "2026-03-28T00:00:00Z",
  "score": 0.82,
  "details": {
    "breakout_price": 172.34,
    "volume_confirmed": true
  },
  "tags": ["vcp-breakout", "volume-confirmed"]
}

Persisted-row contract
----------------------
- Screener results persisted by jobs include provenance fields: `created_by_job_run_id`, `created_at`.
- Persisted schema will be available at `tayfin-screener-api/schemas/persisted_screener_result.json`.

Environment & Configuration
---------------------------
The service respects repo-wide configuration patterns. Example env vars and recommended local values:

- `TAYFIN_SCREENER_API_BASE_URL` — base URL used in examples and tests. Example: `http://localhost:8083`
- `DATABASE_URL` — Postgres URL used by the service (if it reads DB directly). Example: `postgresql://tayfin:password@localhost:5432/tayfin_dev`
- `TAYFIN_HTTP_TIMEOUT_SECONDS` — HTTP client timeout used when calling other services. Example: `10`
- `TAYFIN_CONFIG_DIR` — path to service config files. Example: `./tayfin-screener/tayfin-screener-api/config`

Local test notes
---------------
- When running validator scripts against README examples, export the API base URL so curl examples resolve correctly:

```bash
export TAYFIN_SCREENER_API_BASE_URL=http://localhost:8083
```

- Keep example responses minimal and valid JSON — validators will parse them against JSON Schemas placed under `tayfin-screener-api/schemas/`.

Validation & QA
--------------
- Place JSON Schema files under `tayfin-screener-api/schemas/` with filenames listed above.
- Use the repo-wide README example validator pattern (see `tayfin-indicator/tayfin-indicator-api/scripts/validate_examples.py`) to validate examples against schemas.

Error handling
--------------
- API returns standard JSON error objects with `code`, `message`, and optional `details`.
